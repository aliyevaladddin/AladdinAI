# Scripts

Thin shell wrappers around the commands you'll run most often. Most of these
are also exposed as `make` targets (see the root [`Makefile`](../Makefile));
the scripts are the source of truth and the Make targets just call them.

All scripts assume you run them from the repo root, but each one re-roots
itself to the project before doing anything (`cd "$(dirname "$0")/.."`), so
calling them from elsewhere works too.

---

## `install.sh`

Bootstraps the Python virtualenv at `./.venv` and installs backend deps.

```bash
./scripts/install.sh
# or
make install
```

- Creates `.venv` with `python3 -m venv` if it doesn't exist.
- Upgrades `pip` and installs `backend/requirements.txt`.
- Override the interpreter with `PYTHON=python3.12 ./scripts/install.sh`.

Frontend deps are not handled here — run `cd frontend && npm install`
separately.

---

## `dev-backend.sh`

Starts FastAPI with auto-reload on `0.0.0.0:8000`.

```bash
./scripts/dev-backend.sh
# or
make dev-backend
```

Uses `.venv/bin/uvicorn` directly, so you don't need to activate the venv.
Equivalent to `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000` from
inside `backend/`.

---

## `dev-frontend.sh`

Starts Next.js dev server on `:3000`.

```bash
./scripts/dev-frontend.sh
# or
make dev-frontend
```

Just runs `npm run dev` inside `frontend/`. Make sure you ran `npm install`
there at least once.

---

## `migration.sh`

Generates a new Alembic revision from current model changes.

```bash
./scripts/migration.sh "add agent_triggers table"
# or
make migration m="add agent_triggers table"
```

- Description argument is required.
- Output lands in `backend/alembic/versions/<hash>_<slug>.py`.
- **Always inspect the autogenerated file** before applying — Alembic's
  detection is good but not perfect (especially around server defaults,
  enum changes, and JSON columns).

---

## `migrate.sh`

Wraps `alembic` with the venv path baked in. Defaults to `upgrade head`.

```bash
./scripts/migrate.sh                 # alembic upgrade head
./scripts/migrate.sh downgrade -1    # roll back one
./scripts/migrate.sh history         # any alembic command works
# or
make migrate
make downgrade
```

Errors out with a hint if `.venv/bin/alembic` is missing — usually means you
need to run `install.sh` first.

---

## When to add a new script

Add one when:

- A multi-step command becomes part of the daily workflow.
- A shell incantation has more than two flags worth remembering.
- CI needs to invoke the same logic developers run locally.

Don't add one to wrap a single, obvious command. `npm run build` doesn't
need a wrapper.
